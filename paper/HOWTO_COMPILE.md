# Compiling the paper to PDF (arXiv-ready)

The manuscript is `paper/main.tex`, with:
- `paper/references.bib` — bibliography (verified arXiv IDs)
- `paper/numbers.tex` — auto-generated numeric macros (from `scripts/analyze.py`)
- `paper/tables/*.tex` — auto-generated tables
- `paper/figures/*.png` — auto-generated figures

## Option A — Overleaf (no install, ~1 min)
1. Create a new Overleaf project.
2. Upload everything in `paper/` (keep the `tables/` and `figures/` subfolders).
3. Set the compiler to **pdfLaTeX** and the main document to `main.tex`.
4. Recompile. Done — this is the PDF you upload to arXiv.

## Option B — Local (one command)
Install a TeX engine, then:

```bash
cd paper
latexmk -pdf main.tex          # runs pdflatex + bibtex automatically
# or, with tectonic (single binary):
tectonic main.tex
```

To install an engine on macOS:
```bash
brew install --cask mactex-no-gui      # full TeX (large)
# or, lightweight single binary:
brew install tectonic
```

## arXiv submission
Upload the `paper/` directory as a `.zip` (arXiv compiles LaTeX server-side).
Primary category: `cs.CL` (cross-list `cs.LG`, `cs.AI`). The `references.bib` and
`numbers.tex`/`tables/`/`figures/` are all included, so it compiles as-is.

## Refreshing the numbers
All numeric values in the paper come from `paper/numbers.tex`, regenerated by:
```bash
python scripts/analyze.py
```
so the manuscript always matches `results/`. Never hand-edit numbers.
